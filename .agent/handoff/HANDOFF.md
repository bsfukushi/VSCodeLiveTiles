    素の `SetForegroundWindow` で通る。詰まった相手に付くと自分の入力処理まで止まる危険 API）

  （イベントは専用スレッドから来るので Dispatcher 経由。static イベントなので OnClosed で購読解除）
- `GetWidgetMonitor` / `GetTargetMonitor` は throw をやめて null 返しに。
  モニター 0 枚（RDP 切断中など）の起動クラッシュ経路を封鎖。0 枚時は配置を保留し、
  構成が戻った DisplaySettingsChanged で配置し直す。クリック時に 0 枚なら何もしない
