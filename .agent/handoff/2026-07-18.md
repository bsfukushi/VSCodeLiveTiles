## 解決した問題・知見

- 毎日出ていた 1.1〜1.5 秒の Send 停止は UpdateThumbnailRects が犯人ではない
  （停止時刻に SlowIf 200ms 内訳ログが皆無 / 深夜 3〜4 時集中 → スタンバイ復帰時のページイン濃厚）。
  DWM RPC 抑制は予防的掃除として実施
- 7/15 19:22 の未処理例外は Windows シャットダウン時の .NET ランタイム差し替え（10.0.9→10.0.10）。
  コードの問題ではない
- **明滅の決定論的検証手法**: 合成 `permission_request` を events.jsonl に追記して非前面タイルに
  承認待ちを作成 → リング明滅を目視（同一画素 D03C8F↔AF3579）→ 合成 `session_end` で掃除。
  CLI から任意の CC 状態を作れる
- ハマり: dev ビルド初回が黒画面 → bin 出力の appsettings.json が前セッションの "window" のまま
- ハマり: リリースビルドが MSB3027（検証用に起動したままの dev exe がロック）
